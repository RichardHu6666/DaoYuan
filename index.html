How The System Moves

从代码里能看出，这个项目最核心的价值不是“抓到网页”，而是把异构信息稳定穿过整条链路，最后落成面向学生的可用回答。

Pipeline Narrative
1

Capture

来源进入系统前并不统一，代码里已经明确覆盖公众号文章、教务数据库记录以及网页侧用户资料与日程输入。

2

Normalize

`data_clean` 负责原文清理、字段抽取、去重与校验，让通知不只是原始文本，而是能继续排序和检索的结构化对象。

3

Reason

`decision` 根据用户画像、上下文、候选事件和工具入口选择路径，决定是“直接给入口”还是“整理后回答”。

4

Respond

`if_land_web-fusion` 把上述结果落到网页、聊天窗口和用户配置页，让课表、DDL、竞赛、讲座和办事入口在同一界面里汇合。

Grounded In Code
Question Types 代码和说明里已经明确覆盖“最近有什么竞赛”“今天有什么 DDL”“课表是什么”“教务入口在哪里”等典型校园问题。
Operational Surface 决策层提供 CLI、长期服务与测试入口；清洗层提供单条、批量、重置、远端 LLM 准备等命令。
UX Connection 网页端不是单独演示页，它通过 `/api/chat`、`decision_bridge`、个人画像和课表导入机制与核心后端对接。
